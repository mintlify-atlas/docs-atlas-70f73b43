## createVolume

Creates a disk volume from a disk offering. The volume must be attached to an instance before it can be used.

**Async:** Yes

```
GET /client/api?command=createVolume
```

### Request parameters

<ParamField query="name" type="string">
  The name of the disk volume.
</ParamField>

<ParamField query="diskofferingid" type="uuid">
  The ID of the disk offering. Either `diskofferingid` or `snapshotid` must be provided.
</ParamField>

<ParamField query="snapshotid" type="uuid">
  The ID of a snapshot to create the volume from. Either `snapshotid` or `diskofferingid` must be provided.
</ParamField>

<ParamField query="zoneid" type="uuid">
  The ID of the availability zone in which to create the volume.
</ParamField>

<ParamField query="size" type="long">
  Arbitrary volume size in GB. Use with custom disk offerings.
</ParamField>

<ParamField query="miniops" type="long">
  Minimum IOPS. Use with custom IOPS disk offerings.
</ParamField>

<ParamField query="maxiops" type="long">
  Maximum IOPS. Use with custom IOPS disk offerings.
</ParamField>

<ParamField query="account" type="string">
  The account to create the volume for. Must be used with `domainid`.
</ParamField>

<ParamField query="domainid" type="uuid">
  Domain ID for the volume. If account is not specified, the volume is assigned to the caller's account and domain.
</ParamField>

<ParamField query="projectid" type="uuid">
  Create the volume for a specific project. Mutually exclusive with `account`.
</ParamField>

<ParamField query="virtualmachineid" type="uuid">
  UUID of an instance to attach the volume to after creation. Used together with `snapshotid`.
</ParamField>

### Response fields

<ResponseField name="id" type="string">
  UUID of the created volume.
</ResponseField>

<ResponseField name="name" type="string">
  Name of the volume.
</ResponseField>

<ResponseField name="type" type="string">
  Volume type: `ROOT` or `DATADISK`.
</ResponseField>

<ResponseField name="size" type="long">
  Size of the volume in bytes.
</ResponseField>

<ResponseField name="state" type="string">
  State of the volume: `Allocated`, `Ready`, `Destroy`, `Expunging`, `Expunged`, `UploadOp`, `Migrating`, `Snapshotting`, `RevertSnapshotting`.
</ResponseField>

<ResponseField name="zoneid" type="string">
  UUID of the zone.
</ResponseField>

<ResponseField name="diskofferingid" type="string">
  UUID of the disk offering.
</ResponseField>

<ResponseField name="virtualmachineid" type="string">
  UUID of the instance this volume is attached to, if any.
</ResponseField>

<ResponseField name="created" type="string">
  Creation timestamp.
</ResponseField>

### Example

```bash
curl "http://192.0.2.10:8080/client/api?\
command=createVolume\
&name=data-vol-01\
&diskofferingid=5c1a3d8f-...\
&zoneid=a2f3de84-...\
&size=100\
&apiKey=your-api-key\
&signature=your-signature"
```

## attachVolume

Attaches a disk volume to a virtual machine instance.

**Async:** Yes

```
GET /client/api?command=attachVolume
```

### Request parameters

<ParamField query="id" type="uuid" required>
  The UUID of the disk volume to attach.
</ParamField>

<ParamField query="virtualmachineid" type="uuid" required>
  The UUID of the instance to attach the volume to.
</ParamField>

<ParamField query="deviceid" type="long">
  The device slot ID to map the volume to in the guest OS. If not specified, the next available slot is used. Use `0` to attach as ROOT disk.

  On XenServer, device IDs map to: `0` → `/dev/xvda`, `1` → `/dev/xvdb`, `2` → `/dev/xvdc`, and so on.
</ParamField>

### Response fields

<ResponseField name="id" type="string">
  UUID of the volume.
</ResponseField>

<ResponseField name="name" type="string">
  Name of the volume.
</ResponseField>

<ResponseField name="virtualmachineid" type="string">
  UUID of the instance the volume is now attached to.
</ResponseField>

<ResponseField name="deviceid" type="long">
  Device slot ID used for this attachment.
</ResponseField>

<ResponseField name="state" type="string">
  Current volume state.
</ResponseField>

### Example

```bash
curl "http://192.0.2.10:8080/client/api?\
command=attachVolume\
&id=7f3a1c9e-...\
&virtualmachineid=b2e4f1a0-...\
&apiKey=your-api-key\
&signature=your-signature"
```

## createSnapshot

Creates an instant snapshot of a volume.

**Async:** Yes

```
GET /client/api?command=createSnapshot
```

### Request parameters

<ParamField query="volumeid" type="uuid" required>
  The UUID of the disk volume to snapshot.
</ParamField>

<ParamField query="name" type="string">
  Optional name for the snapshot.
</ParamField>

<ParamField query="quiescevm" type="boolean" default="false">
  If `true`, quiesces the virtual machine before taking the snapshot. Recommended for application-consistent snapshots.
</ParamField>

<ParamField query="asyncbackup" type="boolean" default="false">
  If `true`, the backup operation is asynchronous.
</ParamField>

<ParamField query="locationtype" type="string" default="primary">
  Storage location type for managed storage: `primary` or `secondary`.
</ParamField>

<ParamField query="policyid" type="uuid">
  UUID of a snapshot policy. If not specified, uses `MANUAL_POLICY`.
</ParamField>

<ParamField query="account" type="string">
  Account to create the snapshot for. Must be used with `domainid`.
</ParamField>

<ParamField query="domainid" type="uuid">
  Domain ID for the snapshot.
</ParamField>

<ParamField query="zoneidlist" type="uuid[]">
  Comma-separated list of zone IDs to make the snapshot available in. The snapshot is always available in the volume's zone. Introduced in 4.19.
</ParamField>

<ParamField query="tags" type="map">
  Key/value tag pairs to assign to the snapshot.
</ParamField>

### Response fields

<ResponseField name="id" type="string">
  UUID of the created snapshot.
</ResponseField>

<ResponseField name="name" type="string">
  Name of the snapshot.
</ResponseField>

<ResponseField name="volumeid" type="string">
  UUID of the source volume.
</ResponseField>

<ResponseField name="volumename" type="string">
  Name of the source volume.
</ResponseField>

<ResponseField name="state" type="string">
  State of the snapshot: `Creating`, `CreatedOnPrimary`, `BackingUp`, `BackedUp`, `Copying`, `Error`.
</ResponseField>

<ResponseField name="created" type="string">
  Creation timestamp.
</ResponseField>

<ResponseField name="intervaltype" type="string">
  Snapshot interval type: `MANUAL`, `HOURLY`, `DAILY`, `WEEKLY`, `MONTHLY`.
</ResponseField>

### Example

```bash
curl "http://192.0.2.10:8080/client/api?\
command=createSnapshot\
&volumeid=7f3a1c9e-...\
&name=db-snapshot-$(date +%Y%m%d)\
&quiescevm=true\
&apiKey=your-api-key\
&signature=your-signature"
```

## createTemplateFromSnapshot

Creates a template from an existing snapshot.

**Async:** Yes

```
GET /client/api?command=createTemplate
```

### Request parameters

<ParamField query="name" type="string" required>
  Name for the new template.
</ParamField>

<ParamField query="displaytext" type="string" required>
  Human-readable description of the template.
</ParamField>

<ParamField query="ostypeid" type="uuid" required>
  UUID of the OS type for the template.
</ParamField>

<ParamField query="snapshotid" type="uuid">
  UUID of the snapshot to create the template from. Either `snapshotid` or `volumeid` must be provided.
</ParamField>

<ParamField query="volumeid" type="uuid">
  UUID of the volume to create the template from.
</ParamField>

<ParamField query="ispublic" type="boolean" default="false">
  If `true`, the template is visible to all accounts.
</ParamField>

<ParamField query="isfeatured" type="boolean" default="false">
  If `true`, the template is featured on the template listing.
</ParamField>

<ParamField query="passwordenabled" type="boolean" default="false">
  If `true`, instances deployed from this template support password reset.
</ParamField>

### Example

```bash
curl "http://192.0.2.10:8080/client/api?\
command=createTemplate\
&name=golden-image-v2\
&displaytext=Golden+Image+v2\
&ostypeid=c3f1b8e2-...\
&snapshotid=9a2b4c6d-...\
&apiKey=your-api-key\
&signature=your-signature"
```
