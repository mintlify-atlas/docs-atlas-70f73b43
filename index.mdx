Apache CloudStack is a production-ready, open-source Infrastructure as a Service (IaaS) platform used by hundreds of organizations worldwide to build public, private, and hybrid clouds. It provides compute orchestration, networking, storage management, user accounts, and a full-featured API — all in a single platform.

## Getting started

<Steps>
  <Step title="Install CloudStack Management Server">
    Set up the management server on a supported Linux distribution (Ubuntu or RHEL/CentOS). See the [installation guide](/installation) for full instructions.
  </Step>
  <Step title="Configure a Zone">
    Define a zone, pod, cluster, and add hypervisor hosts. CloudStack supports KVM, VMware vSphere, XenServer, XCP-ng, and Hyper-V.
  </Step>
  <Step title="Add storage and network">
    Configure primary storage for VM disk I/O and secondary storage for templates and snapshots. Set up a network offering for your VMs.
  </Step>
  <Step title="Launch your first VM">
    Use the web UI, CLI, or API to deploy a virtual machine from a registered template. Access it via the console or SSH.
  </Step>
</Steps>

## Supported hypervisors

CloudStack manages workloads across the industry's most popular hypervisors from a single control plane:

| Hypervisor | Use Case |
|---|---|
| **KVM** | Open-source, Linux-native, recommended for new deployments |
| **VMware vSphere** | Enterprise VMware environments |
| **XenServer / XCP-ng** | Xen-based deployments |
| **Hyper-V** | Windows Server environments |
| **OVM / LXC** | Container and Oracle VM workloads |

<Note>
  CloudStack 4.x supports multiple hypervisors within the same cloud deployment. You can run KVM and VMware hosts in different zones within a single CloudStack installation.
</Note>
